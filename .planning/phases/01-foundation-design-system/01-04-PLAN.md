  <files>

  <action>
    **Root cause:** HandleInertiaRequests is global web middleware. SetLocale is route-level middleware applied via routes/web.php. Laravel processes global middleware BEFORE route middleware. So when share() runs, app()->getLocale() still returns the default 'en' because SetLocale hasn't executed yet.

    **Fix 1 — HandleInertiaRequests.php (resolves UAT Tests 2 AND 6):**
    In the share() method, remove the eager `$locale = app()->getLocale()` on line 38. Replace the 'locale', 'direction', and 'translations' values with closure-based lazy evaluation that defers reading until AFTER all middleware (including route middleware) has run:

    ```php
    public function share(Request $request): array
    {
        return [
            ...parent::share($request),
            'name' => config('app.name'),
            'auth' => [
                'user' => $request->user(),
            ],
            'sidebarOpen' => ! $request->hasCookie('sidebar_state') || $request->cookie('sidebar_state') === 'true',
            'locale' => fn () => app()->getLocale(),
            'direction' => fn () => app()->getLocale() === 'ar' ? 'rtl' : 'ltr',
            'translations' => fn () => $this->loadTranslations(app()->getLocale()),
        ];
    }
    ```

    Inertia's share mechanism resolves closures at render time (after all middleware). This is the documented Inertia pattern for lazy evaluation.

    **Fix 2 — app.blade.php (resolves UAT Test 2 font issue):**
    On line 47, the `<body>` tag has `class="font-sans antialiased"`. The `font-sans` class applies Instrument Sans via Tailwind's --font-sans variable, which has HIGHER specificity than the `[dir="rtl"] { font-family: var(--font-arabic); }` rule in app.css (because the class selector on body beats the inherited property from html's dir attribute).

    Fix: Make the font class conditional on locale in the Blade template:

    ```blade
    <body class="{{ app()->getLocale() === 'ar' ? 'font-arabic' : 'font-sans' }} antialiased">
    ```

    NOTE: This Blade expression also suffers from the same middleware ordering issue — app()->getLocale() in the Blade template runs AFTER all middleware INCLUDING route middleware (because template rendering happens inside the response), so this WILL work correctly. The Blade template renders during `$next($request)` inside the middleware stack, which is after SetLocale has run.

    **Fix 3 — app.css (resolves UAT Test 3 backdrop-blur transition):**
    Lines 173-177 define a global transition rule:
    ```css
    html *, html *::before, html *::after {
        transition: background-color 0.3s ease, color 0.15s ease, border-color 0.3s ease;
    }
    ```
    This OVERRIDES Tailwind's `transition-all` utility class because it explicitly lists only 3 properties. When SiteHeader applies `transition-all`, the global rule takes precedence (both have equal specificity, but the global rule is in a later stylesheet position or the cascade order wins). The `backdrop-filter` property is excluded from the transition.

    Fix: Add `backdrop-filter` to the global transition rule:
    ```css
    html *, html *::before, html *::after {
        transition: background-color 0.3s ease, color 0.15s ease, border-color 0.3s ease, backdrop-filter 0.3s ease;
    }
    ```

    **Fix 4 — sheet.tsx (resolves UAT Test 6 secondary issue):**
    On line 73, the SheetPrimitive.Close button uses hardcoded `right-4` for positioning. In RTL layouts, the close button should be on the LEFT side of a left-opening sheet. Change `right-4` to `end-4` (Tailwind's logical property equivalent: right in LTR, left in RTL).

    Find line 73:
    ```tsx
    <SheetPrimitive.Close className="... absolute top-4 right-4 ...">
    ```
    Replace `right-4` with `end-4`:
    ```tsx
    <SheetPrimitive.Close className="... absolute top-4 end-4 ...">
    ```
