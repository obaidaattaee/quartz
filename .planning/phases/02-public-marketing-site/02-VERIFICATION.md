### Gaps Summary

Two infrastructure gaps block the build and prevent full verification:

**Gap 1 — Corrupted Wayfinder file (tsc fails):** The auto-generated `resources/js/routes/newsletter/index.ts` contains duplicate code fragments starting at line 89. The file was apparently regenerated or partially overwritten, leaving an orphaned code block. This file is picked up by `tsconfig.json` and causes 3 TypeScript syntax errors. The file itself is not imported by any phase 2 component (all form submissions use string URL literals), so it does not affect runtime behavior — but `tsc --noEmit` fails. Fix: truncate lines 89–99 from the file (the valid `export default newsletter` is on line 99).

**Gap 2 — Missing radix-ui package (build fails):** The `accordion.tsx` component imports from the `radix-ui` unified package (`import { Accordion as AccordionPrimitive } from "radix-ui"`). This package is declared in `package.json` but is not present in `node_modules`. All other UI components in this project use the scoped `@radix-ui/*` packages (e.g., `@radix-ui/react-collapsible`, `@radix-ui/react-dialog`). The build fails with: "Rolldown failed to resolve import 'radix-ui' from accordion.tsx". This blocks the FAQ page (which uses Accordion) and the entire Vite production build. Fix: run `npm install` to install the missing package, OR change the import in `accordion.tsx` to match the project's established pattern using `@radix-ui/react-accordion`.

These two gaps are entirely infrastructure/dependency issues — all phase 2 page and component code is correctly implemented. Resolving them is a matter of fixing one auto-generated file and installing one npm package.
